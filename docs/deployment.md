# Інструкція з розгортання у Production (Deployment)

Цей документ призначений для release engineer/DevOps фахівців і описує процес розгортання консольного C++ додатку для математичного моделювання у виробничому (production) середовищі.

### 1. Вимоги до апаратного та програмного забезпечення
* **Архітектура:** x64 (64-bit).
* **CPU:** Будь-який сучасний x64 процесор (мінімум 1 ГГц).
* **Оперативна пам'ять (RAM):** від 512 МБ (додаток споживає мінімум ресурсів).
* **Дисковий простір:** 50 МБ вільного простору для зберігання виконуваного файлу та генерації файлу результатів (`tpar.rez`).
* **Операційна система:** Windows 10, Windows 11 або Windows Server 2016 (і новіші).
* **Необхідне програмне забезпечення:** На цільовій машині обов'язково має бути встановлений пакет **Microsoft Visual C++ Redistributable (x64)**.

### 2. Налаштування мережі, серверів та СУБД
* **Мережа:** Додаток працює повністю автономно в режимі офлайн і не потребує налаштування фаєрволів, відкритих портів чи доступу до інтернету.
* **Конфігурація серверів:** Спеціального налаштування вебсерверів (IIS, Nginx, Apache) не вимагається.
* **Налаштування СУБД:** Бази даних не використовуються. Усі результати обчислень серіалізуються локально у файл `tpar.rez`.

### 3. Процес розгортання коду (Deployment)
Розгортання полягає у компіляції Release-версії та її доставці на цільову робочу станцію або сервер обчислень.

1. На машині розробника або на CI/CD сервері (наприклад, за допомогою GitHub Actions) виконайте збірку проєкту у конфігурації **Release**:
   ```bash
   msbuild BachelorProject.sln /p:Configuration=Release /p:Platform=x64
   ```
2. Знайдіть скомпільований виконуваний файл `BachelorProject.exe` у директорії `x64\Release\`.
3. Скопіюйте файл `BachelorProject.exe` на цільовий сервер у робочу директорію, наприклад: `C:\Apps\BachelorProject\`.
4. Переконайтеся, що користувач, від імені якого запускатиметься додаток, має права на запис (Write) у цю директорію (для створення файлу `tpar.rez`).

### 4. Перевірка працездатності (Health Check)
Щоб зрозуміти, що розгортання пройшло успішно і система працює коректно:
1. Відкрийте консоль (cmd або PowerShell) на цільовому сервері.
2. Перейдіть у робочу директорію та запустіть програму:
   ```cmd
   cd C:\Apps\BachelorProject\
   BachelorProject.exe
   ```
3. Програма попросить ввести кількість ітерацій `NN`. Введіть тестове значення, наприклад `10`, і натисніть Enter.
4. **Критерії успіху:** * У консоль будуть виведені розраховані математичні значення `mpar` та `sig_tpar`.
   * Програма завершиться повідомленням `Finish!`.
   * У тій самій директорії успішно створиться (або оновиться) файл `tpar.rez` із результатами розрахунків.
