# Статичний аналіз коду (Linting) та автоматизація

## Обраний лінтер: clang-tidy
Для проєкту мовою C++ було обрано `clang-tidy`. Він підтримується інструментарієм LLVM, тісно інтегрований у MS Visual Studio та має найширший набір перевірок для сучасного стандарту C++.

## Базові правила
Конфігурація зберігається у файлі `.clang-tidy`. Активовані такі групи:
* `readability-*`: покращення читабельності (іменування, магічні числа).
* `modernize-*`: рекомендації щодо використання сучасних конструкцій.
* `bugprone-*`: виявлення патернів, що часто призводять до помилок (наприклад, втрата точності).

## Git Hooks
Налаштовано `pre-commit` хук (розташований у `.git/hooks/pre-commit`). Він інтегрований у процес контролю версій і автоматично запускає перевірку коду перед кожною фіксацією змін.

## Статична типізація (Static Type Checking)
C++ є мовою зі строгою статичною типізацією. Перевірка типів здійснюється на етапі збірки компілятором MSVC. Для посиленого контролю у конфігурації проєкту встановлено максимальний рівень попереджень `/W4`.

## Інтеграція з процесом збірки
Створено скрипт `check.bat`, який об'єднує процес компіляції, перевірки типів та статичний аналіз коду в єдину автоматизовану команду.
