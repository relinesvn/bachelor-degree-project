# Інструкція з оновлення системи (Update & Rollback)

Цей документ описує покроковий процес оновлення консольного додатка `BachelorProject.exe` у виробничому середовищі, а також процедуру відкату (rollback) у разі виникнення критичних помилок.

### 1. Підготовка до оновлення
* **Планування часу простою (Downtime):** Оскільки додаток запускається періодично (для виконання пакетних обчислень), оновлення слід проводити між запусками. Час простою мінімальний (менше 1 хвилини).
* **Перевірка сумісності:** Перед оновленням release-інженер повинен перевірити Release Notes розробників, щоб переконатися, що нова версія не змінює формат серіалізації вихідного файлу `tpar.rez` (або попередити аналітиків про зміну формату).

### 2. Створення резервних копій
Перед будь-якими змінами на сервері необхідно зробити бекап поточної стабільної версії та накопичених даних:
1. Зробити копію поточного виконуваного файлу: 
   ```cmd
   copy C:\Apps\BachelorProject\BachelorProject.exe C:\Apps\BachelorProject\BachelorProject_backup.exe
   ```
2. Зробити резервну копію накопичених результатів розрахунків:
   ```cmd
   copy C:\Apps\BachelorProject\tpar.rez C:\Apps\BachelorProject\tpar_backup.rez
   ```

### 3. Процес оновлення
1. **Зупинка служб:** Переконатися, що процес `BachelorProject.exe` зараз не виконується (перевірити через Task Manager або команду `tasklist | findstr BachelorProject.exe`).
2. **Розгортання нового коду:** Замінити старий файл `BachelorProject.exe` на новий, зібраний із гілки `main` (конфігурація Release).
3. **Міграція даних та оновлення конфігурацій:** Не вимагається, оскільки проєкт не використовує зовнішніх конфігураційних файлів чи СУБД.
4. **Перевірка після оновлення:** Виконати тестовий запуск програми з мінімальною кількістю ітерацій (наприклад, `NN=10`). Переконатися, що нові дані успішно дописуються у файл `tpar.rez`.

### 4. Процедура відкату (Rollback) 
*Ця процедура виконується, якщо нова версія завершується аварійно (Crash) або видає математично некоректні результати.*

1. Видалити нову ("зламану") версію виконуваного файлу.
2. Відновити програму з попередньо створеного бекапу: 
   ```cmd
   move /Y C:\Apps\BachelorProject\BachelorProject_backup.exe C:\Apps\BachelorProject\BachelorProject.exe
   ```
3. Якщо файл даних `tpar.rez` був пошкоджений або перезаписаний новою версією у неправильному форматі, відновити його:
   ```cmd
   move /Y C:\Apps\BachelorProject\tpar_backup.rez C:\Apps\BachelorProject\tpar.rez
   ```
4. Повідомити команду розробки про невдале оновлення із зазначенням кодів помилок або симптомів.
