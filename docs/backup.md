# Стратегія резервного копіювання та відновлення (Backup & Recovery)

Цей документ описує процедури збереження коду та користувацьких даних проєкту `BachelorProject`.

### 1. Стратегія та об'єкти резервного копіювання
Оскільки проєкт є консольним додатком без використання СУБД, резервному копіюванню підлягають:
1. **Вихідний код та конфігурації:** Зберігаються у хмарному репозиторії GitHub.
2. **Користувацькі дані та логи (файл `tpar.rez`):** Містять результати математичного моделювання і зберігаються локально на цільовому сервері.

### 2. Типи, частота та ротація резервних копій
* **Вихідний код:** * *Тип:* Повні та інкрементальні (забезпечується архітектурою Git).
  * *Частота:* При кожному коміті/пуші.
* **Користувацькі дані (`tpar.rez`):**
  * *Тип:* Повні (створення копії всього файлу).
  * *Частота:* Щодня о 02:00 ночі (Daily) та щотижня у неділю (Weekly).
  * *Ротація:* Щоденні копії зберігаються 7 днів. Щотижневі копії архівуються і зберігаються 3 місяці на зовнішньому хмарному сховищі (наприклад, AWS S3 або Google Drive).

### 3. Процедура створення резервних копій (Автоматизація)
Для автоматизації бекапу даних на Windows-сервері використовується вбудований планувальник завдань (Task Scheduler), який запускає наступний `.bat` скрипт:
```cmd
@echo off
set BACKUP_DIR=C:\Backups\BachelorProject
set DATE_STAMP=%date:~-4,4%%date:~-10,2%%date:~-7,2%
mkdir %BACKUP_DIR% 2>nul
xcopy C:\Apps\BachelorProject\tpar.rez %BACKUP_DIR%\tpar_%DATE_STAMP%.rez /Y /I
```

### 4. Перевірка цілісності
Перевірка цілісності виконується автоматично під час резервного копіювання (утиліта `xcopy` повертає код помилки у разі невдачі). Раз на місяць DevOps-інженер вибірково відкриває архівний файл `tpar.rez` у текстовому редакторі для візуальної перевірки формату даних (`NN=... mpar=... sig_tpar=...`).

### 5. Процедура відновлення (Disaster Recovery)
У разі пошкодження або втрати файлу `tpar.rez`:
1. Зупинити виконання програми `BachelorProject.exe`.
2. Перейти до директорії з бекапами (`C:\Backups\BachelorProject\`).
3. Знайти останній валідний файл за датою (наприклад, `tpar_20260324.rez`).
4. Скопіювати його в робочу директорію (`C:\Apps\BachelorProject\`) та перейменувати на `tpar.rez`.
5. Зробити тестовий запуск програми (`NN=1`), щоб переконатися, що нові дані успішно дописуються у відновлений файл.
