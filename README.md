# Моделювання флуктуацій температури пари

Цей репозиторій містить вихідний код бакалаврського проєкту. 
Програма написана на C++ і виконує математичне моделювання флуктуацій температури пари з урахуванням випадкових впливів (газ, навантаження, вприск) за допомогою операції згортки.

## Вимоги
* Microsoft Visual Studio (C++)

## Запуск
1. Відкрийте `.sln` файл у Visual Studio.
2. Скомпілюйте та запустіть програму.
3. Результати обчислень будуть збережені у файл `tpar.rez`.

## 🏗 Архітектура проєкту

Проєкт є автономним (standalone) консольним додатком для математичного моделювання, розробленим мовою C++.
* **Вебсервер / Application server / Кешування:** Відсутні (не передбачені предметною областю).
* **СУБД (База даних):** Відсутня. Зберігання результатів моделювання виконується локально.
* **Файлове сховище:** Локальна файлова система комп'ютера (результати записуються у файл `tpar.rez`).
* **Діаграма архітектури:** `Користувач -> [CLI Інтерфейс] <-> [Модуль математичної згортки] -> [Файлова система (tpar.rez)]`

## 🛠 Інструкція для розробників (Developer Onboarding)

Цей розділ допоможе вам швидко налаштувати середовище та запустити проєкт на "чистій" ОС Windows.

### 1. Необхідне програмне забезпечення та залежності
* **Git:** Для клонування репозиторію ([Завантажити Git](https://git-scm.com/)).
* **Microsoft Visual Studio:** Версії 2019 або 2022 (Community, Professional або Enterprise).
  * *Важливо:* Під час встановлення Visual Studio обов'язково оберіть робоче навантаження **"Розробка класичних програм на C++" (Desktop development with C++)**. Це автоматично встановить необхідний компілятор MSVC та стандартні бібліотеки. Інших сторонніх залежностей проєкт не має.

### 2. Клонування репозиторію
Відкрийте термінал (PowerShell або командний рядок) та виконайте команду:

```bash
git clone https://github.com/relinesvn/bachelor-degree-project.git
cd bachelor-degree-project
```

### 3. Налаштування середовища та запуск у режимі розробки
1. Відкрийте файл рішення `BachelorProject.sln` подвійним кліком (він автоматично відкриється у Visual Studio).
2. У верхній панелі інструментів переконайтеся, що обрано конфігурацію **Debug** та платформу **x64**.
3. *Примітка:* База даних для цього проєкту не потрібна, тому додаткова конфігурація підключень не вимагається.
4. Натисніть **F5** (або кнопку "Локальний відлагоджувач Windows") для збірки та запуску програми.
5. У консолі програма попросить ввести кількість ітерацій `NN`. Введіть ціле число (наприклад, `1000`) і натисніть Enter.

### 4. Базові команди та операції (CLI)
Якщо ви віддаєте перевагу роботі з командного рядка, відкрийте **Developer Command Prompt for VS** і використовуйте такі команди:

* **Збірка проєкту:**
  ```bash
  msbuild BachelorProject.sln /p:Configuration=Debug
  ```

* **Очищення проєкту (видалення скомпільованих файлів):**
  ```bash
  msbuild BachelorProject.sln /t:Clean
  ```

## 📝 Правила документування коду (Doxygen)
Цей проєкт використовує **Doxygen** для автоматичної генерації документації з коду. Будь-хто, хто долучається до розробки, повинен дотримуватись наступних правил:
1. **Коментарі Javadoc-style:** Усі публічні функції, класи та глобальні константи повинні мати блок коментаря, що починається з `/**`.
2. **Обов'язкові теги:** * `@brief` — короткий опис того, що робить сутність.
   * `@param` — опис кожного вхідного параметра.
   * `@return` — опис того, що повертає функція.
3. **Розширені теги (для складної логіки):** Використовуйте `@details` для опису математичних алгоритмів (наприклад, згортки) та `@warning` для потенційних помилок (наприклад, ділення на нуль або від'ємна дисперсія).
